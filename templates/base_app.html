{# ---------- Sidebar ---------- #} {% comment %} Left nav. Width-animated collapse on click of the hamburger in the topbar — uses the ``sidebar`` Alpine store (persisted to ``localStorage``). When closed: ``w-0 overflow-hidden`` so children stay in the DOM (no re-init on reopen) but take zero space. {% endcomment %} {% comment %} No ``overflow-hidden`` on the aside — that would clip the icon-rail tooltip when the sidebar is collapsed. Child elements either use ``truncate`` (single-line names) or are hidden via ``x-show`` while collapsed, so we don't need the parent to clip overflow horizontally. The inner ``nav`` keeps its own ``overflow-y-auto`` scroll. {% endcomment %} {% comment %} ``hx-boost`` intercepts every anchor inside the sidebar and routes it through HTMX instead of a full page reload. ``hx-select`` plucks ``#app-content`` out of the response — the server detects boosted requests via ``HX-Boosted: true`` and returns the **full** template (extends base_app) so the select target exists. Result: only the right-hand column swaps; the sidebar element stays mounted, its Alpine state (collapse / theme / selection) and Lucide-rendered icons survive across navigations. {% endcomment %}
