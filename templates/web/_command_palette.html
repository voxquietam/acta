{% load i18n %} {% load lucide %} {% comment %} Global command palette (Cmd/Ctrl+K). Mounted once in ``base_app.html`` so it overlays every authenticated page. Self-contained Alpine component — no esbuild bundle. Behaviour: * Toggled by ``Cmd+K`` / ``Ctrl+K`` anywhere (ignored while typing in an input/textarea/contenteditable — handled in acta.js so it shares ``isTypingTarget`` with the other shortcuts). * Listens to ``acta:palette-open`` and ``acta:palette-toggle`` window events for the topbar button trigger. * Fetches ``web:palette_search?q=…`` with a 120 ms debounce; the first fetch fires on open so the panel isn't empty. * Keyboard nav: ↑ / ↓ moves a flat cursor over the visible items (sections are skipped), Enter follows the active item's URL, Esc closes. * Click on an item navigates (htmx-boosted swap into ``#app-content`` to match the sidebar nav so Alpine state survives). Items carry ``url`` so the markup is the same for tasks / projects / nav — the section-specific bits (slug pill, project name, icon) sit inside each row template. {% endcomment %}
{% comment %} Hidden icon library — client-side actions don't run through the server, so the Alpine code needs a way to render Lucide glyphs. We pre-render the small set used by ``task_actions`` here and the JS ``iconHtml()`` helper looks them up by name. Keeps icons inline SVG (no FOUC, no extra fetch) and matches everything else in the palette. {% endcomment %}
