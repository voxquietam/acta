Carries its own ``id`` so the toggle endpoint can swap just this piece without disturbing the outer ``#labels-cell`` Alpine state — that keeps the dropdown open across consecutive selections. Two grouped renderings, chosen by ``trigger_layout``: * ``"column"`` (default for the full task page's right rail) — one row per group: fixed-width uppercase caption on the left, chips wrap on the right. Caption ``min-w`` keeps chips aligned across rows when group names differ in length. * ``"row"`` (task detail modal) — every group sits as a self-contained cluster in a single wrapping flex row. Reads cluster boundaries via a larger horizontal gap between clusters than between chips inside one. Single-bucket workspaces (every label in one group, or all Ungrouped) collapse to a flat chip row in either layout — no axis to communicate. {% endcomment %}
