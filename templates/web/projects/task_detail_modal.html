Served from ``TaskDetailView`` when the request carries ``?modal=1`` — the row click in tables / kanban cards fires an ``hx-get`` to this URL and swaps the result into ``#modal-root``. Full-page rendering still works on direct URL load (no ``?modal=1``) so shared links open as a page. The modal hosts the same body partial the full page uses (``_task_detail_modal_body.html``) plus a compact header. The body's SSE + inline-edit wiring works unchanged: it relies on element ids / ``data-*`` attrs that are identical in either layout. Behaviour: * The modal is a pure overlay — it never touches the address bar. Opening it swaps into ``#modal-root``; the URL stays on the page behind it.
