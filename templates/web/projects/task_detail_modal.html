{% load i18n %} {% load lucide %} {% comment %} Modal-mode rendering for the task detail view. Served from ``TaskDetailView`` when the request carries ``?modal=1`` — the row click in tables / kanban cards fires an ``hx-get`` to this URL and swaps the result into ``#modal-root``. Full-page rendering still works on direct URL load (no ``?modal=1``) so shared links open as a page. The modal hosts the same body partial the full page uses (``_task_detail_body.html``) plus a compact header. The body's SSE + inline-edit wiring works unchanged: it relies on element ids / ``data-*`` attrs that are identical in either layout. Behaviour: * Backdrop click → close. * ``Esc`` → close. * Expand button (top-right) → ``history.replaceState`` to the bare task URL and reload, so the user lands on the full-page chrome. * Close pops one history entry, returning to whatever page hosted the click (HTMX's ``hx-push-url`` on the trigger added that entry). {% endcomment %}
