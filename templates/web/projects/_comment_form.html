{% load i18n %} {% comment %} Comment posting form — same TipTap rich editor used for descriptions (``[data-description-editor]``), wired in "comment" mode: * ``data-no-autosave`` skips the blur autosave and pagehide beacon — comments require an explicit submit, half-typed text shouldn't fly off on focus loss or tab close. * The hidden ``input[name="body"]`` carries the markdown the editor serializes on every keystroke (via ``onUpdate`` -> ``data-editor-output``). * The form's ``submit`` button stays disabled while the editor is empty. The editor dispatches an ``editor:change`` custom event on every update that the surrounding Alpine form listens to. * After a successful POST, the editor is cleared via ``commands.clearContent()`` so the form is ready for the next comment without a page reload. {% endcomment %}
