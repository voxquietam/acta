{% load i18n %} {% load lucide %} {% comment %} Selection bubble for the TipTap editor: a floating formatting bar shown over a non-empty text selection. Hidden by default (``display:none``); description_editor.js shows + positions it from the selection's coords on ``onSelectionUpdate`` and hides it on collapse / blur / scroll. Drop this partial inside any element marked ``[data-description-editor]``. Third take, done without a positioning library. The two earlier attempts failed for the same reason — both leaned on tippy: a hand-rolled bubble (7 patches to hide reliably — see ``feedback_stop_kostyling``), then the official ``@tiptap/extension-bubble-menu`` (portals the toolbar to ````, which broke our Alpine link-mode and didn't render). This one keeps the toolbar in our DOM and positions it via ``coordsAtPos`` — no tippy, no portal, so there's nothing fragile to rot. Buttons keep editor focus via mousedown preventDefault and reach the editor via ``closest('.description-editor-toolbar')._editor`` (set in JS). Buttons use ``data-tooltip="…"`` + ``aria-label="…"`` instead of the native ``title`` attribute — the browser default tooltip is black/ yellow OS chrome that clashes with our themed surfaces. The themed tooltip rule in ``main.css`` reads ``data-tooltip`` and renders a card-coloured pop on hover; ``aria-label`` keeps screen-reader accessibility. Two modes via Alpine state — "menu" (default toolbar) and "link" (an inline URL input replacing the buttons, so we don't fall back to the native ``window.prompt`` dialog on link insertion). {% endcomment %}
