{% load i18n %} {% load lucide %} {% load web_extras %} {% comment %} Bulk context-menu fragment — the multi-select twin of ``_task_context_menu.html``. Same chrome + in-place submenu swap, but the header is a live "N tasks" count (from the ``selection`` store) and every action applies to the whole selection via the bulk endpoint (``actaBulkPatch`` / ``actaBulkDelete``) rather than per-task forms. Opened by a right-click on a task that's part of a 2+ selection, and by the bulk bar's "Actions" button. Pickers are populated from the active workspace (not a task). Expects ``status_labels``, ``priority_labels``, ``size_values``, ``workspace_members``, ``workspace_projects``, ``workspace_labels``. The selection ids live client-side in ``$store.selection``. {% endcomment %}
