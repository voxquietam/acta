{% load i18n %} {% load lucide %} {% load web_extras %} {% comment %} Shared task-filter sidebar. Used by All Tasks, My Work, and the per-project Kanban / Table page. Layout: a fixed-right floating panel on lg+. Height = content, capped at viewport. **Sections are collapsible** — each section header is a button that toggles its body via Alpine ``x-show``; the open / closed state is persisted per section in ``localStorage`` (key ``acta:filter-sec:``). When collapsed, the header surfaces an active-count chip + a comma-separated preview of the first two active values, so the user doesn't have to expand to see what's on. Long sections (Labels) carry a **type-to-filter** input above the chip list — Alpine binds an ``:class`` expression on each chip that hides non-matching values without a server round-trip. Section order: search → status → priority → project → labels → workspace. Project sits above labels so the long scrollable list doesn't push the small sections off the fold. Done is a regular status checkbox; the page decides whether done is hidden by default via ``default_show_done`` passed to ``apply_task_filters``. Section visibility flags: * ``filter_hide_assignee`` — drop Assignee (handled by the top strip). * ``filter_hide_project`` — drop Project (page is scoped or hoisted). * ``filter_hide_status`` — drop Status (kanban columns already group by it). Quick Presets, Saved Views, prose-summary line, and collapsed icon rail are queued in [[project-todo-filter-sidebar-presets-views]] (stage B + C of the design rollout). {% endcomment %}
{% comment %} Collapsed-state trigger and expanded form are both always in the DOM; visibility is driven by the aside's ``acta-filters-open`` / ``acta-filters-closed`` class (toggled in :class via the Alpine store). Rules live in main.css. This avoids the HTMX-swap race where Alpine's ``x-show`` directive briefly fails to bind on the re-swapped form, leaving both visible at the same time. {% endcomment %} {% comment %} ``overflow-hidden`` keeps inner content from bleeding past the rounded corners into adjacent columns (My Work / All Tasks lay the filter sidebar next to the task list — without the clip, header text could overlap the list). The tooltips on header buttons compensate with ``data-tooltip-pos="below"`` so they drop DOWN into the form body where they're visible, instead of UP past the clipped edge. {% endcomment %}
