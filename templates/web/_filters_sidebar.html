{% load i18n %} {% load lucide %} {% load web_extras %} {% comment %} Task-filter sidebar v2 — rail (48px) + per-section popover + expanded (512px, two-column) layout. Shared by All Tasks, My Work, and the per-project Kanban / Table page. Architecture ============ One ``
`` owns every input, regardless of which layout is currently visible. Each filter section is rendered ONCE inside ``.acta-flt-content`` (so no duplicate hidden inputs / no form ambiguity). The element carries ``data-mode`` and ``data-open`` attributes that drive CSS-only visibility: * ``[data-mode="rail"][data-open="status"]`` → 288px popover, only the ``[data-section="status"]`` card visible. * ``[data-mode="expanded"]`` → 512px panel, 2-col grid, every section visible at once. State lives in ``$store.filters`` (acta.js): ``mode`` persists to ``localStorage``, ``openSection`` is ephemeral. The legacy ``open`` flag is still mirrored for any caller that hasn't migrated. Tri-state chips & rows ====================== Each chip / row carries Alpine ``x-data="{ state: 'none' | 'included' | 'excluded' }"``. The hidden input's ``name`` flips between the include key (``status``) and the exclude key (``xstatus``). Left-click cycles ``none → included → none``; right-click cycles ``none → excluded → none``. The global ``acta:filter-reset`` event drops every chip back to ``'none'`` so the Reset button works without a hard reload. Cycle is the one exception — chips are include-only. Section visibility flags from the view: * ``filter_hide_assignee`` — drop Assignee (top strip handles it). * ``filter_hide_project`` — drop Project (page is scoped to one). * ``filter_hide_status`` — drop Status (kanban groups by it). {% endcomment %}
