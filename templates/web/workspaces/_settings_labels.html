{% load i18n %} {% load lucide %} {% comment %} Labels & label-groups management card on the workspace settings page (Workflow tab). Wrapped in ``#workspace-labels`` so every CRUD endpoint returns this partial and HTMX swaps it in place; the inline edit / create forms collapse via a single Alpine ``openId`` so only one form shows at a time. Drag-drop is bound by ``acta.js`` (``[data-sortable-labels]`` → POST ``reorder_labels`` on drop). Visual: inner group cards stay transparent (no second-tier ``bg-background`` holes against the parent ``bg-card``) — just a soft border. Section dividers inside each group are thin and few; chips and color dots carry the visual rhythm instead of border lines. Context (built by ``_labels_section_context`` in ``apps/web/views.py``): workspace — current :class:`Workspace` label_groups — list of ``{"group": LabelGroup, "labels": [Label]}`` ungrouped_labels — labels with ``group=None`` label_colors — tuple of curated hex strings (the picker palette) Each label carries an ``usage_count`` annotation (M2M ``tasks`` distinct count) set by the same context helper — N+1-safe. {% endcomment %}
